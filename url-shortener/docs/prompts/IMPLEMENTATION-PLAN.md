# Создание плана реализации из дизайна

Создай пошаговый план реализации на основе дизайна проекта.
Разбей на маленькие фичи с микро-задачами.

Исполнитель — опытный разработчик, но не знает ничего о нашем проекте.

## Процесс

**Прочитай дизайн:**
- Прочитай `docs/prompts/PROJECT-DESING.md` перед началом

**Исследуй технологии из дизайна:**
- Проверь актуальные версии библиотек и лучшие практики
- Найди официальную документацию
- Найди примеры реализации и туториалы
- Узнай про известные проблемы и подводные камни

**Разбей на фичи:**
- Каждая фича — законченная, работающая функциональность
- Фичи идут в логическом порядке
- Одна фича = одна сессия агента

**Для каждой фичи опиши:**
- Цель — что должно работать после выполнения
- Файлы — что создать, что изменить, какие тесты написать
- Шаги — микро-задачи, чеклист `- [ ]`
- Проверка — точная команда с ожидаемым результатом

**Ключевые принципы:**
1. Описывай план кратко и понятно без лишнй воды, при этом раздели на короткие понятные секции
2. Каждый вопрос, где ты не уверен или есть разные решения ты должен описывать кратко мне и ожидай подтверждения от меня.

**Сохрани план:** `docs/results/IMPLEMENTATION-PLAN.md`

## Формат плана

```markdown
# [Название проекта] — План реализации

**Цель:** [одно предложение — что строим]

## Процесс выполнения

**Изучи контекст:**
- Найди первую незакрытую фичу в плане

**Выполни фичу:**
- Выполни все шаги фичи
- Тесты обязательны для каждой фичи

**Подведи итог:**
- Напиши короткий итог: что сделал, что работает, есть ли проблемы
- Жди подтверждения пользователя
- После подтверждения: отметь шаги `- [x]` и поставь ✅ в заголовке фичи

## Принципы

- **DRY** — не дублируй код, выноси общее
- **YAGNI** — не добавляй то, что не описано в дизайне

---

### Фича 1: [Название]

**Цель:** что должно работать после выполнения

**Файлы:**
- Создать: `src/agent/tool-registry.ts`
- Изменить: `src/agent/index.ts`
- Тесты: `tests/agent/tool-registry.test.ts`

**Шаги:**
- [ ] Создать модели Tool и Result
- [ ] Реализовать ToolRegistry с тестами
- [ ] Интегрировать в основной модуль агента

**Проверка:**
- [ ] `pnpm test -- tests/agent/tool-registry.test.ts` — все тесты проходят

---

### Фича 2: [Название]

...
```
